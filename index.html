Теоретические вопросы

  1. Какие есть способы объявления функций?

    2. Ответ:

    • Function Declaration
    • Function Expression
    • Стрелочные функции (arrow functions)

  2. Приведите примеры вызова одной и той же функции всеми известными вам способами.

    3. Ответ:

    1. function sum(a, b) {
      return a + b;
      }
      console.log(sum(2, 3));
    2. function sum(a, b) {
      console.log(a + b);
      }
      sum(10, 2);
    3. function sum(a, b) {
      console.log(a + b);
      }
      let func = sum;
      func(10, 3);
    4. const sum = (a, b)=> a + b;
      console.log(sum(1,2));

  3. В чём разница между тестированием и отладкой (дебаггингом)? А что такое логирование?

    4. Ответ:

    Тестирование - это испытание кода (программы) для выявления ошибок. Отладка, или debugging, — это поиск (локализация), анализ и устранение ошибок в программном обеспечении, которые были найдены во время тестирования.
    В простейшем случае логирование сводится к файлу на диске, куда разные программы записывают (логируют) свои действия во время работы.
    Такой файл называют логом или журналом. Как правило, внутри лога одна строчка соответствует одному действию.
    Самый простой способ, без использования специальных программ для логирования, - это писать в console.log все свои действия последовательно и смотреть на результат.

  4. В чём разница между Function Expression и Function Declaration?

    5. Ответ:

    Если функция объявлена, как отдельная инструкция в основном потоке кода, то это Function Declaration. Если функция была создана как часть выражения, то считается, что эта функция объявлена при помощи Function Expression.

  5. Что делает функция console.log()?

    6. Ответ:

    console.log() — это метод, предназначенный для печати в консоль браузера.

  6. По какой причине, мы можем использовать до объявления функцию, объявленную через Function Declaration, но не можем ту, которая объявлена через Function Expression ?

    7. Ответ:

    Все дело в том КОГДА создается функция.
    Function Expression создается в тот момент, когда выполнение кода доходит до нее, и только потом она уже может использоваться.
    С Function Declaration все иначе. Ее можно использовать во всем скрипте. Потому что, когда движок JS только готовится выполнять код, прежде всего он ищет в нем Function Declaration, и создает все такие функции.

  7. Что такое глобальная зона видимости функций?

    8. Ответ:

    Это область видимости вне любых функций. Переменные, объявленные в ней, видны везде.

  8. Что вернёт код?

    function showX(x) {
    return x;
    }
    console.log(showX(28));

    9. Ответ: 28

  9. Что выведет этот код?

    10. function sum(x, y=0) {
    return x+y;
    }
    console.log(sum(5));

    Ответ: 5

  10. Что выведет код?

    11. console.log(foo);
    let foo = "Cat";
    console.log(foo);

    Ответ:

    Uncaught ReferenceError: can't access lexical declaration 'foo' before initialization. Невозможно получить доступ к лексическому объявлению «foo» перед инициализацией.
